# 実装メモ — n8nコンテナの起動・管理画面アクセス確認

対象タスク(MS1): 「見つけた構成でn8nコンテナを起動し、管理画面にブラウザでアクセスできることを確認する」

## なぜ「実装待ち」なのか

Dockerコンテナの起動・ブラウザでの動作確認はホスト環境(brainリポジトリの外)への操作であり、Claude Codeのheadless実行がbrain内で完結させる範囲を超える。[CLAUDE.md](../../CLAUDE.md)のガードレール「実装(コードを書く)タスクの場合、brainの外は触らない」に準じ、実機作業を伴うこのタスクも人間またはproject側セッションでの実行に委ねる。

## 何をどう実装するか

[30_Tech_Notes/n8n_Docker_Compose最小構成.md](../../30_Tech_Notes/n8n_Docker_Compose最小構成.md) にまとめた構成をそのまま使う。

1. 作業用ディレクトリを用意する(brain外。例: `~/work/n8n/`)
2. そこに以下の `compose.yml` を配置する

   ```yaml
   services:
     n8n:
       image: docker.n8n.io/n8nio/n8n
       restart: unless-stopped
       ports:
         - "127.0.0.1:5678:5678"
       environment:
         - N8N_HOST=localhost
         - N8N_PORT=5678
         - N8N_PROTOCOL=http
         - GENERIC_TIMEZONE=Asia/Tokyo
         - TZ=Asia/Tokyo
       volumes:
         - n8n_data:/home/node/.n8n

   volumes:
     n8n_data:
   ```

3. 起動する

   ```bash
   docker compose up -d
   ```

4. ブラウザで `http://localhost:5678` にアクセスし、初回セットアップ画面(オーナーアカウント作成画面)が表示されることを確認する
5. 確認できたら、この`実装メモ.md`に結果(成功/失敗、詰まった点)を追記し、`tasks.md`の該当行を`- [→]`から`- [x]`に戻す

## 前提・注意点

- 実行環境はRaspberry Pi(StockWatchと同じホスト)を想定。n8n公式イメージはarm64対応だが未検証のため、初回起動時にアーキテクチャ起因のエラーが出ないか確認する
- ポートは`127.0.0.1`限定で公開しているため、リモートから確認する場合はSSHポートフォワーディング等が必要
- `docker`コマンドは**未インストールと確認済み(2026-07-19)**。先にインストールが必要:
  `curl -fsSL https://get.docker.com | sudo sh` → `sudo usermod -aG docker sloasea`(グループ反映に再ログイン)。
  sudoパスワードが必要なためユーザー作業([user-actions.md](../../90_System/user-actions.md) に登録済み)
