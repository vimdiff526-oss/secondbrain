次のタスクを1件実行してください。対象: $ARGUMENTS

## 手順

1. **対象タスクの選定**
   - 引数でプロジェクト指定があればその tasks.md の未完了先頭タスク
   - 指定がなければ pm-layer エージェントで全プロジェクトをスコアリングし、最優先の1件を選ぶ
2. **実行**(brain内で完結する作業のみ)
   - 調査 → 結果を `30_Tech_Notes/` や `50_Business_Context/` にノート化
   - 文書 → 対象ノートを作成・更新
   - 確認 → 結果をREADMEの作業ログに記録
   - **実装(コードを書く)タスクの場合**: brainの外(他リポジトリ)は触らない。代わりに `20_Projects/<名>/実装メモ.md` に「何をどう実装するか」を具体化して書き、タスクを「実装待ち(人間またはproject側セッションで実行)」として `- [ ] → - [→]` に変える
3. **完了処理**: tasks.md のチェックを `- [x]` にし、READMEの作業ログ節に「日付・やったこと・成果物パス」を1行追記する
4. コミット・プッシュする(例: `work: <タスク名>`)

## ガードレール

- 1回の /work で実行するのは**1タスクのみ**。30分相当を超える場合は途中で区切り、続きを新タスクとして tasks.md に足す
- ブロッカー(情報不足・外部待ち)に当たったら、タスクに「ブロック: <理由と解除条件>」を追記して次善のタスクに切り替える(質問で終わらない)
- ファイル削除・force push・brain外の変更は禁止
