次の対象をマイルストーンとタスクに分解してください: $ARGUMENTS

## 手順

1. 対象プロジェクトを `20_Projects/` から特定する。存在しなければ `/new-project` の手順で先に立ち上げる
2. knowledge-explorer エージェントで関連する既存知識・過去の判断を収集する
3. 3層に分解する
   - **ゴール**: READMEの完了の定義を確認(なければ起草)
   - **マイルストーン**: 2〜4個。各MSは「達成したと外から分かる状態」で書く
   - **タスク**: 各MSにつき2〜5個。**1タスク30分以内**の粒度。種別(調査/文書/実装/確認)と
     **実行主体**(`〔Claude〕`= brain内で完結 / `〔ユーザー〕`= 実機操作・対話・意思決定が必要)を必ず添える
4. `tasks.md` にチェックリスト形式で書く:

```markdown
## MS1: <達成状態>
- [ ] 調査〔Claude〕: 〜を調べて30_Tech_Notesにまとめる
- [ ] 実装〔ユーザー〕: 〜(実機作業)
```

   `〔ユーザー〕` タスクがMSの先頭に固まると自動実行が即停止するため、`〔Claude〕` で先に進められる並びを意識する

5. pm-layer エージェントの観点で点検する: 最初のタスクは**15分以内に着手・完了できる粒度**か。大きければ割る
6. コミット・プッシュする(例: `decompose: <プロジェクト名>`)

## ガードレール

- タスク管理の正は tasks.md(GitHub Issues は未連携)
- 既存の tasks.md がある場合は上書きせず、完了済み `- [x]` を保持したまま再構成する
