description: 新しいプロジェクトを20_Projects/に立ち上げる
argument-hint: <プロジェクト名> [ゴールの説明]
---

新しいプロジェクトを立ち上げてください: $ARGUMENTS

## 手順

1. knowledge-explorer エージェントで、このテーマに関連する既存ノート(過去の構想・調査・判断)を探す
2. `20_Projects/<プロジェクト名>/README.md` を作成する
   - frontmatter(`created`, `tags`, `status: active`)
   - 構成: ゴール(1文)/ なぜやるか / 完了の定義 / 関連ノート(手順1の結果をパス付きで)/ 作業ログ(空の節)
3. `20_Projects/<プロジェクト名>/tasks.md` を空のチェックリストとして作成する(タスク分解は `/decompose` の仕事)
4. コミット・プッシュする(例: `project: <名前> 立ち上げ`)

## ガードレール

- 同名・同趣旨のプロジェクトが既にあれば新規作成せず、既存側のREADMEを更新して報告する
- ゴールが引数から読み取れない場合も止まらず、関連ノートから最も妥当なゴールを起草し「ゴールは仮置き」と明記する
